# 구현 계획서 - PG 데이터 자동화 파이프라인 (간소화 버전)

## 1. 개요 (간소화된 워크플로우)
사용자가 전처리(Parsing) 로직을 수동으로 담당하고, 안티그래비티는 수십만 건의 데이터를 1초 만에 병합하는 **통합 대사 연산**에 집중하는 구조입니다.

- **입력:**
  - **[NEW]** 로우 데이터 폴더: `(GL)결제취소원장_주문서/상품 - [연월].xlsx`
  - P 데이터 폴더: 수동 가공이 완료된 PG 기준 데이터
- **출력:**
  - **[NEW]** 마스터 WEV 데이터: 입금처별로 분리된 `WEV_GL_[입금처]_[연월].xlsx` (자동 생성)
  - 통합 엑셀 파일: `w`, `p`, `c(대사결과)` 3개의 시트를 포함하는 `[샵]_[PG명]_[연월]_대사완료.xlsx`

## 2. 신규 추가: 원장 생성기 (`generate_wev_files.py`)
1. **데이터 로드**: `(GL)결제취소원장_상품` 및 `주문서` 엑셀 파일을 읽어옵니다.
2. **그룹화 및 합산**: 주문번호/마감일 기준으로 상품 금액 및 배송비/캐쉬 등을 합산합니다.
3. **병합 (Full Join)**: 상품 데이터와 배송비 데이터를 주문번호 기준으로 병합합니다.
4. **최종 금액 산출**: `원장금액 = (상품 + 관세 + 배송비) - 캐쉬` 공식을 적용합니다.
5. **파일 분할**: 결과 데이터를 **입금처(PG명)**별로 분리하여 `WEV` 폴더에 저장합니다.

## 3. 핵심 로직: `merge_and_report.py`
1. **비교 키 생성**: `마감구분/거래구분 + 주문번호` 조합의 복합 키 사용.
2. **데이터 집계**: 원본 합계 보존을 위해 행 제거 없이 전체 합산(Sum) 수행.
3. **컬럼 스키마 (최종)**:
   - **추가 정보**: `W_마감구분`, `W_결제통화`, `P_거래구분`, `P_요청통화2` 노출.
   - **금액 헤더**: `W_원장금액_결제통화/원화`, `W_주문금액_요청통화/원화`로 명칭 표준화.
4. **UI 최적화**: 엑셀 TEXT 서식, 자동 열 너비, 첫 행 틀 고정 적용.

## 4. 최종 검증 결과 (2026-01 대규모 데이터)
- **처리 규모**: 약 190만 건 (PAYPAL 90만 건 포함)
- **정확도**: GMT(99.9%), NHN(99.5%) 대사 일치율 달성.
- **정밀도**: 원본 파일과 1원 단위까지 100% 일치 확인.
