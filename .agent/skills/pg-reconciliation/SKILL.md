# PG 정산 대사 스킬 (PG Reconciliation Skill)

이 스킬은 사용자가 이미 공통 양식으로 전처리해 둔 PG 데이터(p)와 내부 WEV 데이터(w)를 입력받아 양방향 금액 대사를 수행하고, 최종 결과를 엑셀 파일로 출력(Export)하는 프로세스를 정의합니다.

## 1. 업무 프로세스 개요

대사 작업은 총 2개 페이즈(Phase)로 단순화되어 진행됩니다:
1. **Phase 1: 마스터 기준값 생성 (Deduplication)** - 짝을 이루는 WEV(w) 파일과 PG(p) 파일의 기준값(Key)을 추출하고 병합하여 중복 없는 마스터 리스트(c 시트의 기반)를 만듭니다.
2. **Phase 2: 대사 및 리포팅 (Join & Sum)** - 생성된 마스터 기준값을 중심으로 양쪽 데이터를 조인(Join)하여 합계를 비교하고 차액을 계산한 뒤, `w`, `p`, `c` 세 개의 시트를 가진 최종 엑셀 리포트 파일을 생성합니다.

## 2. 에이전트 행동 지침 (SOP)

사용자가 대사 작업을 요청하면 에이전트는 다음 단계를 순서대로 수행합니다.

### Step 1: 대상 파일 파악 (입력값)
- 사용자가 w 원본 파일 폴더와 p 원본 파일 폴더의 경로를 제공합니다.
- 동일한 파일명(예: `WEV_[샵]_[PG명]_[연월].xlsx`, `PG_[샵]_[PG명]_[연월].xlsx`)을 가진 (w 파일, p 파일) 쌍을 찾습니다.

### Step 2: 대사 파이프라인 실행
- 매칭된 w 파일과 p 파일 쌍에 대해 `execution/merge_and_report.py` 도구를 호출합니다.
- **입력:** w 파일 경로, p 파일 경로, 출력(Output) 폴더 경로
- **수행 내용:** 
  - w, p 데이터 로드 및 기준값 통합/중복 제거 (c 리스트 생성)
  - 기준값 단위 Join 및 Sum 처리 (`[w_금액]`, `[p_금액]`, `[차액]`, `[비교결과]` 컬럼 생성)
  - 최종 엑셀 파일 저장 (생성될 시트명: `w`, `p`, `c`)
- **출력:** `[원본파일명]_대사완료.xlsx`

### Step 3: 결과 보고
- 대사가 완료된 엑셀 파일들의 결과 경로를 사용자에게 각각 안내합니다.
- 총 처리 건수, 일치 건수, 불일치(차액 발생) 건수를 간략히 요약하여 보고합니다.

## 3. 필수 표준 컬럼 매핑 규칙 (Schema Mapping Rules)
사용자가 제공하는 w 파일과 p 파일은 아래의 규칙에 따라 비교됩니다.

### [기준값 (Key) 생성 규칙]
- **W(WEV) 데이터 Key**: `마감구분` + `주문번호` (Concat)
- **P(PG) 데이터 Key**: `거래구분` + `주문번호` (Concat)
*(주의: 주문번호는 엑셀에서 숫자가 깨지지 않도록 출력 시 TEXT 포맷으로 엄격하게 유지해야 합니다.)*

### [금액 대사(Amount) 비교 규칙]
금액은 **결제통화**와 **원화**, 총 두 가지 항목을 동시에 병렬 비교합니다.
1. **결제통화 비교**: 
   - W: `원장금액_결제통화` vs P: `합계 : 주문금액(요청통화)2`
   - 결과값 도출: `차액_결제통화`
2. **원화 비교**:
   - W: `원장금액_원화` vs P: `합계 : 주문금액(원화)2`
   - 결과값 도출: `차액_원화`

## 4. 실행 도구 (Execution Scripts)
- `execution/merge_and_report.py`: 지정된 W와 P 데이터를 읽어와 위 명시된 Key 결합 및 두 가지 금액(결제통화, 원화) 조인 대사를 수행하고, 엑셀 3개 시트 구조 생성을 담당하는 핵심 통합 엔진입니다.
