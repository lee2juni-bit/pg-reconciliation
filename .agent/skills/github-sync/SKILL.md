# GitHub Sync Skill

이 스킬은 로컬 작업 환경의 변경 사항을 GitHub 원격 저장소와 자동으로 동기화합니다.

## 사용 시점
- 새로운 기능을 구현하고 이를 GitHub에 반영하고 싶을 때
- 현재까지의 작업 진행 상황을 안전하게 백업하고 싶을 때
- 사용자가 "내 결과물을 GitHub에 푸시해줘"라고 요청할 때

## 작업 흐름
에이전트는 이 스킬을 호출받으면 다음 단계를 수행합니다:

1. **상태 확인**: `git status`를 통해 변경된 파일이 있는지 확인합니다.
2. **커밋 메시지 결정**: 사용자가 메시지를 제공했다면 해당 메시지를 사용하고, 그렇지 않다면 작업 내용에 기반한 적절한 메시지를 생성합니다.
3. **스크립트 실행**: `execution/github_sync.py` 스크립트를 호출하여 동기화를 수행합니다.

## 실행 명령 예시
```bash
python3 execution/github_sync.py "feat: 한글 학습 게임 캐릭터 선택 기능 추가"
```

## 주의 사항
- 이 스킬은 현재 체크아웃된 브랜치에 그대로 푸시합니다.
- 원격 저장소와 로컬의 상태가 달라 `push`가 거절되는 경우, 에이전트는 무리하게 `force push`를 하지 않고 사용자에게 상황을 보고합니다.
